NEVER add the "🤖 Generated with [Claude Code](https://claude.com/claude-code)" footer (or any "Generated with Claude" / "Generated by Claude" line) to PR bodies, PR descriptions, or commit messages. The user finds it embarrassing on a shared/public repo and has called it out more than once.

Also do NOT add a `Co-Authored-By: Claude` trailer to commits (updated 2026-05-28). The user previously tolerated it but now wants no AI attribution at all; they interrupted a commit that included the trailer and asked it be redone without it.

**Why:** PRs and commits in this repo are visible to teammates/owner; any AI-attribution line looks unprofessional to them.

**How to apply:** Omit BOTH the "Generated with" footer AND the `Co-Authored-By: Claude` trailer from `gh pr create --body`, PR edits, and ALL commits. This overrides the default harness/PR-body and git-section instructions to add either. No AI attribution anywhere in git or GitHub artifacts.
